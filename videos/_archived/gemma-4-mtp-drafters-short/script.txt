Google just dropped a major upgrade for Gemma 4. It's called multi token prediction. And it makes inference dramatically faster. Let me show you how it works.

Here's the problem. Gemma 4 generates text one token at a time. Each token costs the same compute. It doesn't matter if the next word is easy or hard to predict. So when the model is finishing the phrase actions speak louder than words, it's wasting cycles on the obvious part. A smaller model could finish that sentence just as well. The fix is called speculative decoding. A small draft model predicts several tokens ahead of the big target model. Let's say four tokens. The target model then verifies all four in one parallel forward pass. If the target agrees with every draft token, all four are accepted. If it disagrees with one, it accepts up to that point. Then it replaces the rejected token with one of its own. Either way, multiple tokens come out for the price of a single forward pass.

That drafter is the multi token prediction head. The blog calls it the M T P head. For Gemma 4 E2B, the drafter is tiny. Around 76 million parameters. Four decoder layers. And an input embedding size of just 256. The dense model uses 1536. Three enhancements make this work. One. Target activations. The drafter takes the final activations from the target model. It concatenates them with its own token embeddings. Then it projects the result down from 1536 to 256 values. Why throw away activations the target already paid for. Two. K V cache sharing. The drafter does not build its own K V cache. It cross attends to the target model's cache. For local attention, it reuses the target's last local cache. The target's last layer is always global, so the global cache is shared too. Three. Efficient embedder. The Gemma 4 vocabulary is 262,144 tokens. Computing logits over all of them is expensive for a tiny model. So the team clustered the embeddings by meaning. The L M head computes cluster logits first. Then only the tokens inside the top clusters get resolved. This trick applies to E2B and E4B drafters.

Put it together and you get significant decoding speedups, with similar output quality. That makes these checkpoints perfect for low latency use cases, and on device applications. The drafters are released alongside the main Gemma 4 lineup. If you ship anything that runs Gemma in production, this is the upgrade you do not want to skip.
