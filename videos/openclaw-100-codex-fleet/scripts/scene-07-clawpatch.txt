It's called claw-patch dot A I. Same pattern. Open source, MIT, made by the OpenClaw team. The trick is how it slices a repo before review.

Most AI code review tools see one giant codebase and choke. Claw-patch maps the repo into semantic units first. Routes for Next dot J S. Commands for N P M or Go or Rust or Swift. Packages. C L I scripts. Tests. Each unit gets its own entrypoints, owned files, context files, and associated tests. Then the review runs per unit, not per file.

The findings come back categorized. Bug, security, performance, docs gap, test gap, maintainability. Each one tagged with a severity and a confidence score and pointed at the exact lines of evidence. You get a markdown report and a JSON report. Clean handoff to the next agent.
