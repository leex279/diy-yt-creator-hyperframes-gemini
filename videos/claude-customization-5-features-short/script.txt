Most developers use one. But Claude Code actually has five distinct customization features. Each solves a different problem.

First, CLAUDE dot M D. Always-on instructions that load into every session.

Second, Skills. On-demand expertise that loads only when relevant.

Third, Sub-agents. Isolated workers that run in their own context window.

Fourth, Hooks. Event-driven automation that fires on specific triggers.

And fifth, MCP Servers. External tool integrations via an open protocol.

Five questions tell you which one to pick. Should Claude always know it? CLAUDE dot M D. Should Claude know it sometimes? Skills. Should it run in isolation? Sub-agents. Should it happen automatically on events? Hooks. Does Claude need external tools? MCP Servers.
