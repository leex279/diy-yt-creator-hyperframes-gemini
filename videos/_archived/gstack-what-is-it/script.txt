g stack hit eighty-nine thousand stars in a few weeks. So — what is it? And do you need it?

Garry Tan, the CEO of Y Combinator, open-sourced it earlier this year. Today it has eighty-nine thousand GitHub stars and over three hundred pull requests.

At its core, g stack is a pack of slash-commands for Claude Code. Specialist roles instead of a blank prompt. All markdown. All free.

Twenty-three specialist skills. Eight power tools. Twenty-eight commands in total — covering planning, design, code review, browser testing, and shipping the pull-request.

The most-talked-about skill is office-hours. It asks six forcing questions about your product before you ship a single line of code.

So — does it work? Some founders say yes. They report shipping faster, getting un-stuck on architecture, and using office-hours to invalidate ideas before they waste a weekend on them. The praise is real.

Others are skeptical. They ask: is this really new — or is it Claude Code's existing plan and review commands wrapped up with nicer ergonomics? How is it different from BMAD or Spec Kit?

And critics push back further. At the end of the day, they say — these are markdown files. Stars on a repo aren't the same as production value.

So — we're not telling you what to think. We're asking. Have you tried g stack? What are you using to ship faster — g stack, BMAD, Spec Kit, superpowers, or just raw Claude Code? Drop it in the comments.
