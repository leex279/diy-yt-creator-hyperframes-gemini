Building videos with code usually means React. State. JSX. Build steps. HyperFrames takes a different path. It is just HTML. Plus GSAP. That is the whole stack.

Here is what you get out of the box. One CLI. Lint. Preview. Render. Zero config to start. Sub-compositions for scene splits. A built-in registry for shared blocks. Determinism baked in. No randomness. No clocks. No network fetches at render time.

Three rules to remember. Number 1. Every timed element needs data-start and data-duration. That is how the renderer knows when each clip enters and exits. Number 2. Add class equals clip for visibility control. The framework hides anything outside its time window. Number 3. Register your GSAP timeline on window dot double-underscore timelines. The renderer drives playback from there.
