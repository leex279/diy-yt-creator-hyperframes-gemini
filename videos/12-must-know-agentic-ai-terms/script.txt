Twelve words run the entire agent stack. Get three of them wrong, and your demo ships, but your production agent burns money, leaks secrets, and confidently lies. Here is the right map. By the end of this video, you will read any 2026 agent paper, doc, or RFP and translate vendor speak without ambiguity. And by the way, three of these you use wrong every week. Watch for which.

Four roles. Twelve words. Read each one once. Context reaches outward to the world. Reasoning thinks. Team coordinates. And Control stops bad things from happening. The trick is not memorizing the list. The trick is knowing which quadrant your problem sits in. Because the fix for a hallucination is not the fix for a security leak. And the fix for a slow agent is not the fix for a confused one.

Let us start with Context. Three words. M C P. Tool Use. Memory. M C P is the Model Context Protocol. Anthropic released it on November 25th, 2024. Think U S B C for A I. One standard. Any model. Any tool. By early 2026 the registry has over 10,000 servers. Tool Use is the older sibling. OpenAI shipped function calling on June 13th, 2023. The model returns a schema validated function call. You execute it. You feed the result back. The two are not the same thing. M C P is the transport and the catalog. Tool Use is the language the model uses to ask. And Memory is the third leg. The persistent layer across sessions. MemGPT proved in late 2023 that an L L M plus three tiers of memory beats a bigger context window. Core memory in R A M. Recall memory in vector storage. Archival memory in cold storage. Mem0 took that further. In 2026 they published numbers showing 91 percent lower p95 latency and 90 percent fewer tokens versus stuffing everything in the prompt.

Now Reasoning. Three more words. Agent Loop. Grounding. Context Window. The Agent Loop is the cycle. Perceive. Plan. Act. Observe. The ReAct paper from Princeton and Google, arXiv 2210.03629, October 2022, is where it started. They proved that interleaving reasoning and action beats either one alone, by 34 points on one benchmark and 10 on another. Reflexion built on that and hit 91 percent pass at one on HumanEval. Every agent framework you use today, every Claude Code session, every Cursor invocation, runs some version of this loop. Grounding is what stops the model from making things up. The Anthropic Citations A P I shipped on June 23rd, 2025. One customer, Endex, dropped hallucination rates from 10 percent to zero. You give the model documents. The model writes the answer. Every sentence carries a span back to the source. Context Window is the token budget per inference. Today we are at 1 million tokens on Claude. But more is not always better. The Lost in the Middle paper, Liu and team 2023, arXiv 2307.03172, proved it. Information in the middle of a long context gets ignored. The model attends to the start and the end. So you can have a million tokens and still miss the line that mattered.

Now Team. Three words. Orchestrator. Subagent. Multi Agent. The Orchestrator is the primary agent. It decomposes the goal. It delegates. Anthropic documented their pattern in June 2025. One lead off agent. Two to four subagents on a normal task. Ten or more on complex research. Each Subagent owns one task with its own context window and its own tool set. They run in parallel. They report back. The Orchestrator synthesizes. The receipt. Anthropic measured a 90.2 percent improvement on their internal research benchmark over a single agent. The cost. 15 times more tokens than a single chat. Multi Agent is the system property. Not a single thing. The collaboration of two or more specialized agents. AutoGen from Microsoft, August 2023, arXiv 2308.08155, is the canonical paper. The trade is always the same. More agents means more capability and more cost. The question is never which is better. The question is which problem needs which shape.

Now Control. Three words. Guardrails. Sandboxing. Human in the Loop. Three layers. They each fail differently. Guardrails are inference time rules. N Vidia NeMo Guardrails ships five rail types. Input. Dialog. Retrieval. Execution. Output. Plus Colang, a small language for writing them. Constitutional A I from Anthropic, December 2022, is the research backbone. Guardrails fail when the rule is not written. Sandboxing is the environment. The agent runs untrusted code. You isolate it. Firecracker, the AWS micro V M, was open sourced in November 2018. 125 millisecond boot. Under 5 megabytes of R A M overhead. The M I T A I Agent Index from 2025 was blunt. Docker alone is insufficient for untrusted agent code. E2B, one of the dominant sandbox vendors, grew from 40,000 sandbox sessions a month to 15 million in one year. Sandboxes fail when the escape exists. Human in the Loop is the approval gate. Claude Code ships 6 discrete permission modes. Ask. Auto. Bypass. Plan. Accept Edits. Deny. The catch. 93 percent of permission prompts get approved. Humans rubber stamp. Human in the Loop fails when the human stops paying attention.

Now the four arguments your team is going to have. M C P versus Tool Use. M C P is the protocol. Tool Use is the message. Different layer. Different scope. Context versus Memory versus R A G. Context is what fits in this inference. Memory is what persists across them. R A G is one way to populate either. Guardrails versus Sandboxing versus Human in the Loop versus Permissions. Guardrails are rules. Sandboxes are environments. Humans are gates. Permissions are configurations. Different controls for different failures. Subagent versus Orchestrator versus Multi Agent. Subagent is a worker. Orchestrator is a coordinator. Multi Agent is the system. One thing. Another thing. The whole thing. Resolve those four and you are done arguing about agent architecture for the rest of the year.

The decision tree. Five questions. Pick the first yes. Question one. Did your agent hallucinate or forget. The answer lives in Grounding, Memory, or Context Window. Question two. Could it not reach the data it needed. That is Tool Use or M C P. Question three. Did it do something destructive. Guardrails. Sandboxing. Human in the Loop. Question four. Was one agent overloaded with a goal too big for it. Orchestrator. Subagent. Multi Agent. Question five. Did it answer in one shot when the problem needed iteration. That is the Agent Loop. Five questions. Twelve concepts. Every production bug maps somewhere.

That is the stack. Twelve words. One cycle. Four roles. Context reaches. Reasoning thinks. Team coordinates. Control stops. Anchor every term to the phase where it fires, and you will never confuse vocabulary with architecture again.
